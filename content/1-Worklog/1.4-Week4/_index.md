### Week 4 Objectives:

* Learn about serverless architecture with Lambda and how to build event-driven functions on AWS.
* Practice integrating Lambda with API Gateway to build RESTful APIs.
* Get familiar with CI/CD automation for Lambda/API Gateway using GitHub Actions, CodeBuild, and CodePipeline.
* Understand the role of IAM Role/Policy when integrating AWS services and ensuring pipeline security.

### Week 4 Achievements:

* Gained a clear understanding of serverless architecture with AWS Lambda: event-driven execution, no infrastructure management, auto scaling, and pay-per-invocation model.
* Practiced creating Lambda functions, compared runtimes, understood handler, execution environment, timeout/memory limits, and how to debug logs with CloudWatch.
* Explored common Lambda triggers: direct test, API Gateway, S3, etc.
* Learned about API Gateway: distinguished REST API vs HTTP API, understood resources, methods, mapping templates, CORS configuration.
* Practiced integrating Lambda with API Gateway, created RESTful endpoints, tested with Postman, understood request/response flow, common errors (timeout, mapping, missing IAM permissions) and troubleshooting.
* Understood the CI/CD model on AWS: automated pipeline for build, test, deploy Lambda/API Gateway.
* Practiced connecting GitHub Actions with AWS, using CodeBuild for build/test, CodePipeline for automated deployment.
* Recognized the role of IAM Role/Policy when integrating AWS services, and how to grant the minimum permissions needed for a secure pipeline.
* Documented practical steps, real-world errors, and solutions when implementing CI/CD for Lambda/API Gateway.
