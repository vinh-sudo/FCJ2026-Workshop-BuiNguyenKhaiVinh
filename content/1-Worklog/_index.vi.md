**Tổng quan worklog:**

Worklog này ghi lại quá trình thực tập và phát triển hệ thống quản lý, thống kê sản xuất sử dụng các dịch vụ AWS, AI, CI/CD, tối ưu vận hành thực tế. Tổng thời gian thực hiện: 12 tuần (từ 06/01/2026 đến 03/04/2026).

### Danh sách các tuần và nội dung chính:

- **Tuần 1:** [Làm quen với AWS, tạo tài khoản, thiết lập VPC, nhóm, bảo mật, vẽ kiến trúc mẫu, thực hành các thao tác cơ bản](1.1-Week1/)
- **Tuần 2:** [Tìm hiểu, thực hành VPC, EC2, Security Group, IAM, SSH, phân quyền, kiểm soát truy cập](1.2-Week2/)
- **Tuần 3:** [Mua domain, tích hợp Route 53, hybrid DNS, Load Balancer, best practice thiết kế hệ thống DNS](1.3-Week3/)
- **Tuần 4:** [Xây dựng Lambda, tích hợp API Gateway, CI/CD với GitHub Actions, CodeBuild, CodePipeline, IAM role/policy](1.4-Week4/)
- **Tuần 5:** [Chốt ý tưởng, proposal, thực hành ECR/ECS, xây dựng pipeline CI/CD, triển khai ứng dụng](1.5-Week5/)
- **Tuần 6:** [Secrets Manager, S3, thiết lập database, migration, tích hợp backend, bảo mật, quản lý tài nguyên](1.6-Week6/)
- **Tuần 7:** [S3, IAM role, phân tích nghiệp vụ, vai trò, quy tắc, ôn tập dịch vụ backend AWS](1.7-Week7/)
- **Tuần 8:** [Thiết lập/tối ưu database, cấu hình backend (CORS, JWT, Redis), nghiệp vụ manager](1.8-Week8/)
- **Tuần 9:** [Xây dựng chức năng plan, dashboard cho manager, kiểm thử API, hiểu quy trình nghiệp vụ quản lý](1.9-Week9/)
- **Tuần 10:** [Schedule tự động, pause/resume order, upload S3, notification SNS/SQS, tối ưu code, clean code](1.10-Week10/)
- **Tuần 11:** [SES, quên mật khẩu, AI Q&A, report, tính toán progress, backup/restore dữ liệu Railway-RDS](1.11-Week11/)
- **Tuần 12:** [Triển khai thực tế ECR/ECS, CloudWatch, tối ưu chi phí, CI/CD, review tổng thể hệ thống](1.12-Week12/)
